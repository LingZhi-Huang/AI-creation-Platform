### 📝 

#### 一、后端考核项目概述

开发一个 AI 创作与算力租赁平台。用户可以通过消耗 “算力点数（Tokens）” 来提交 AI 生成任务（如文本对话、AI 绘图等）。系统需要处理高并发的任务提交、异步生成调度、算力点数充值支付以及订单补偿机制。以下黄色标注接口，大二同学必做，大一同学作为扩展加分要求。

#### 二、技术栈

* 基础框架：Spring Boot

* 数据库：MySQL + MyBatis/MyBatis-Plus

* 缓存：Redis

* 认证：JWT

* 消息队列：Kafka / RabbitMQ

* 支付：支付宝沙箱环境

* 文档 / 测试：Apifox 文档

* * *

#### 三、用户端

##### 1. 认证模块

* 登录接口（和注册功能合并一个接口）
  
  * 密码登录
  
  * 邮箱登录（后续购买成功使用邮箱发送通知）

* 退出登录

##### 2. 用户模块

* 获取当前用户信息

* 修改个人信息

* 签到领取算力点数（利用 Redis 实现每日签到和防刷）

##### 3. AI 创作中心模块

* 作品画廊 / 对话广场
  
  * 分页查询平台上公开的优秀 AI 生成内容
  
  * 支持条件筛选（按热度、分类）

* 高并发任务提交
  
  * 用户提交一段 Prompt（提示词）请求生成
  
  * 使用 Redis 校验并扣减用户的算力点数（需处理并发安全，防止超扣）
  
  * 接口防刷与限流（例如同一 IP 10 秒内只能提交 1 次请求）

* 异步任务调度
  
  * 点数扣减成功后，将生成任务推送到消息队列（MQ），立刻给用户返回 “任务排队中”
  
  * 消费者监听队列，调用大模型 API 获取数据，随后更新数据库中该任务的状态为 “已完成”。

* 结果通知
  
  * 生成完成后，通过邮箱发送通知，或是提供接口供前端轮询任务状态。

##### 4. 算力充值与订单模块

* 分页查询用户的充值订单记录。

* 创建充值订单（购买 100 算力包、月卡等），获取支付宝二维码

* 支付回调处理：验证支付宝签名，更新订单状态，给用户账户增加对应算力。

* 掉单补偿：定时任务 / 延迟队列去支付宝查单，防止回调丢失导致的支付成功但未加算力。

* 超时关单：创建订单 30 分钟未支付，自动关闭订单。

* * *

#### 四、管理端

##### 1. 基础功能（复用普通用户登录接口）

* 登录

* 退出登录

##### 2. 用户管理

* 分页查询用户信息

* 查询用户详情

* 修改用户信息

* 启用 / 禁用用户

* 手动给用户补偿 / 扣除算力点数

##### 3. 算力商品与模型管理

* 添加充值套餐

* 上架充值套餐（添加并不等于上架）

* 下架充值套餐

* 管理系统支持的 AI 模型选项

* 调整每次调用的算力消耗单价

##### 4. 任务与订单管理

* 查询全站的 AI 生成任务流水（查看排队中、成功、失败的任务）

* 订单管理：查询充值订单，处理异常订单（手动补单）
